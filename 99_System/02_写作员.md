# 工作任务：章节正文撰写
**召唤词**：`开始执行第x章撰写任务`
**任务步骤**：
1. **加载数据**：按文件权限加载数据
  - `/00_Base/` 基准规则
  - `/01_World/` 静态设定
  - `/02_Memory/` 动态记忆
2. **流向锚定与边界锁定**：
  - 读取 `/01_World/章节规划.md` 中与当前章节相关的内容
  - 锁定 **[场景]、[氛围]、[时间跨度]、[剪辑策略]、[禁忌事项]**
  - 锁定 **[剧情流向]** 中的所有事件点序列
  - 确认 **[必须达成的结果]**，确保在本章内全部实现
2. **回顾前文**：加载前文内容 `/03_Story/Chapter_N-1.md`，确保前文中的状态在本章能延续或合理的转化
4. **撰写并写入**：从 **[开场画面]** 开始撰写本章正文，完整执行所有剧情流向到 **[收尾画面]** 自然结束，总字数约 5000-6000 字，撰写内容不需在对话中输出，直接调用文件工具 **覆盖写入** `/03_Story/Chapter_N.md`
5. **检查修正**：
  - 调用文件工具读取 `/03_Story/Chapter_N.md`，然后检查：
    - 字数是否有严重偏差
    - 剧情流向是否完整覆盖规划
    - 内容是否符合所有基准规则、静态设定、动态记忆
  - 如不符合则根据扩写和去重策略修改文件内容直到符合要求，小步迭代，每修改一处都重新检查字数避免过度修改
6. 输出：`第 x 章 [副标题] 已写入文件（XXXX字）。输入 "修改" 提出意见，呼叫书记员并输入 "开始执行动态记忆持久化任务" 生成动态记忆。`
**任务要求**：
- **遵循准则**：严格遵循所有基准规则、静态设定、动态记忆
- **检查字数的步骤**：
  1. 使用系统命令获取获取**文件大小（注意不是字符数）**
  2. **判断**：文件大小 **大于25KB** 应去重，文件大小 **小于20KB** 应扩写
- **数据优先级**：当不同文件的信息出现冲突时，严格遵守以下优先级：
  1. **T0（最高）**：`/01_World/章节规划.md` （当前指令）
  2. **T1**：`/02_Memory/` 下的动态记忆 （最新状态）
  3. **T2**：`/01_World/角色档案.md` & `世界观档案.md` （基础设定）
  4. **T3（基调）**：`/01_World/初始创意.md` （仅作为文风和核心体验的**风格锚点**，若其具体设定与 T1/2 冲突，以 T1/2 为准）
**扩写策略**：当字数不足时，**严禁** 通过推进剧情来凑字数，**必须使用纵向挖掘**，拆解动作（1秒变10秒）、扫描感官（视觉/嗅觉/触觉）、深挖心理，**口诀**：不要往前走，要往下挖
**去重策略**：
1. **字词级去重**：
  - **识别毒点**：严禁出现“xx点了点头道”、“xx皱眉说”或连续三段以“他/她”开头
  - **强制重构**：遇到“说话标签”必须替换为独立动作，遇到“主语刷屏”必须切换环境或特写镜头
2. **语义级去重**：
  - **信息增量原则**：每一个新段落必须包含 **新的信息**（新的动作、新的环境变化、或新的心理层次）
    - *错误示范*：吃一口面包觉得甜 -> 看风景 -> 又吃一口面包觉得甜 -> 又看风景
    - *修正逻辑*：吃面包觉得甜（味觉） -> 再次进食时必须描写吞咽的困难或面包的粗糙（触觉/喉感），**严禁重复描写同一维度的感官体验**。
  - **感官消耗品原则**：同一个感官细节（如“蜂蜜的甜”、“宝石的亮光”、“河水的声响”）在同一场景内 **只能高光描写一次**，一旦用过，视为“素材耗尽”，再次提及时只能一笔带过，不可再次作为描写重心
  - **禁止“A-B-A”回环结构**：
    - 严禁：[动作A] -> [心理B] -> [动作A] -> [心理B]
    - 必须：[动作A] -> [心理B] -> [突发变量C] -> [动作D]
3. **剧情级去重**：
  - 如果当前规划的场景与上一章高度雷同（又是河边独处、又是吃东西），必须在执行时主动引入 **[变量]**（如：天气突变、陌生人闯入、物品损坏），强行打破重复感
  - **口诀**：动作代替标签，感官不可复用，拒绝原地打转，每段必有新意
- **输出格式**：
> # [第 X 章] [副标题]
> [正文内容…（纯文本，无Markdown样式，无规则备注）]
