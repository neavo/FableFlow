# 任务步骤：
1. **明确目标章节**：
  - 用户如输入具体章节数，则目标章节为用户输入的章节
  - 如果用户没有输入，则检查 `03_Story/` 中的文件列表，最新章节 +1 即为目标章节，没有文件则为第 1 章
2. **加载数据**：按文件权限加载数据
  - `00_Base/` 基准规则
  - `01_World/` 静态设定
  - `02_Memory/` 动态记忆
3. **流向锚定与边界锁定**：
  - 读取 `01_World/章节规划.md` 中与当前章节相关的内容
  - 锁定 **[场景]、[氛围]、[时间跨度]、[剪辑策略]、[禁忌事项]、[必须达成的结果]**
  - 锁定 **[剧情流向]** 中的所有事件点序列
4. **回顾前文**：加载前文内容 `03_Story/Chapter_N-1.md`，确保前文中的状态在本章能延续或合理的转化
5. **分块撰写**：
  - 从 **[开场画面]** 开始，按场景块逐块撰写逐块写入文件，直到 **[收尾画面]** 自然结束
  - 撰写时 **严格遵循任务要求**
  - 每块字数约 1500 字（中文），执行对应的剧情流向事件点序列
  - 每块文本直接调用文件工具 **覆盖/追加写入** `03_Story/Chapter_N.md`，不需在对话中输出
6. 每个场景块完成时输出：`第 X 场景块 已撰写完毕。`，全部完成后输出：`第 x 章 [副标题] 已撰写完毕。`

# 任务要求：
- **遵循准则**：严格遵循所有基准规则、静态设定、动态记忆
- **数据优先级**：当不同文件的信息出现冲突时，严格遵守以下优先级
  1. **T0（最高）**：`01_World/章节规划.md` （当前指令）
  2. **T1**：`02_Memory/` 下的动态记忆 （最新状态）
  3. **T2**：`01_World/角色档案.md` `世界观档案.md` （基础设定）
  4. **T3（基调）**：`01_World/初始创意.md` （仅作为文风和核心体验的**风格锚点**，优先级最低）
- **深刻挖掘**：当剧情已经写完了但字数未达标时，严禁推进到下一剧情，必须原地深挖
  - **显微镜**：将动作描写拆解
  - **联想闪回**：由物体触发一段的微型回忆
  - **环境动态**：描写背景环境的一个微小变化
- **避免重复**：
  - **字词级**：
    - **识别毒点**：严禁"xx 点了点头道"、"xx 皱眉说"或连续三段以"他/她"开头
    - **强制重构**：说话标签替换为独立动作，主语刷屏时切换环境或特写镜头
  - **语义级**：
    - **信息增量**：每段必须包含新信息（新动作、环境变化或心理层次），禁止重复同一感官维度的体验
      - ❌ 吃面包觉得甜 → 看风景 → 又吃面包觉得甜
      - ✅ 吃面包觉得甜（味觉） → 再次进食时描写吞咽的困难或粗糙口感（触觉）
    - **感官消耗**：同一感官细节（甜味、亮光、声响）在场景内只能高光描写一次，再次提及需一笔带过
      - 示例：首次详细描写"蜂蜜的甜"，第二次只需"甜意还在"，不可再次展开
    - **线性推进**：禁止 A-B-A 回环结构，必须动作 → 心理 → 新变量 → 新动作
      - ❌ 走路（动作）→ 累（心理）→ 继续走路（动作）→ 更累（心理）
      - ✅ 走路（动作）→ 累（心理）→ 发现新路（新变量）→ 停下观察（新动作）
  - **章节间**：
    - **感官素材锁定**：上一章已高光的感官细节，本章需跳过描写或切换全新感官维度
      - 示例：上章详细写"河水的声响"，本章直接写"河水流淌"或改写"冰凉的触感"
    - **动作重构**：相似动作必须同时改变句式、描写角度和时间节奏
      - 示例：上章"他拿起杯子喝了一口" → 本章改为"杯子举到嘴边，液体滑过喉管"
    - **跨章增量**：连续相似场景必须展现不同身体感受、心理层次和细节聚焦
      - 示例：连续三章战斗：第一章写疼痛，第二章写麻木，第三章写心理变化
- **输出格式**：
  > # [第 X 章] [副标题]
  >
  > [正文内容…（纯文本，无额外样式）]
